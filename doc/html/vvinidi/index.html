Data conversion
For converting data within the file formats dicom, nifti and vista you can use the tool vvinidi included in the lipsia-sandbox package. The only exception is writing of file format dicom. That is not possible at this time.

Usage of vvinidi
The usage of vvinidi is just as straightforward as you can imagine. For simple conversion you only have to type the input image file and the output image file.
For instance, the conversion from type nifti into type vista looks just like:

> vvinidi -in input.nii -out output.v

Special cases you have to consider
Using FSL

If you are using FSL and want to convert an image of any type into format nifti, you have to attach the parameter -wdialect fsl.

> vvinidi -in #image of any type# -out output.nii -wdialect fsl

The parameter -wdialect denotes the dialect used for writing the image. Equally there is a parameter -rdialect which refers to the dialect used to read an image.

Reading functional vista data

Normally vvinidi tries to detect whether the image contains functional data or anatomical data. But unfortunately, there are some cases which do not fit in the pattern of detection, particularly when reading functional vista images.
So if the automatic detection of the functional vista image is not successful you can attach the parameter -rdialect functional.

> vvinidi -in functional.v -out output.nii -rdialect functional

Specify repetition time

vvinidi gives you the chance to set the repetition time (in seconds) of your output image by attaching the parameter -tr.

> vvinidi -in input.nii -out output.v -tr 3.5

Using vvinidi as a data type conversion tool
Consider you just want to convert the data type of an image. That is, changing the pixel representation (e.g. from short into float) but retaining the image format. This can be achieved by adding the parameter -repn representation.

> vvinidi -in input.nii -out output.nii -repn float

Of course you can convert the pixel representation and the image format in one step.

> vvinidi -in input.nii -out output.v -repn float

Reporting bugs
Because image conversion is sort of a life task it is almost impossilbe to provide for all contingencies. So if you take notice of something odd regarding the tool vvinidi and its output do not hesitate to report this either to tuerke@cbs.mpg.de or reimer@cbs.mpg.de. But please be sure your input image data is not corrupted prior to blame the data conversion itself. Thanks!


Parameters of 'vvinidi':
-help
Prints usage information.
-in
Input file. Default: (none)
-out
Output file. Default: (none)
-rdialect
Dialect for reading. The available dialects depend on the capabilities of IO plugins.
-wdialect
Dialect for writing. The available dialects depend on the capabilities of IO plugins.
-repn
Representation in which the data shall be written.
Options are: {boolean,color,color,double,dvector4,float,fvector4,ivector4,s16bit,s32bit,s64bit,s8bit,u16bit,u32bit,u64bit,u8bit}
-tr
Repetition time in s.
-rf
Override automatic detection of file suffix for reading with given value.
-wf
Override automatic detection of file suffix for writing with given value.
