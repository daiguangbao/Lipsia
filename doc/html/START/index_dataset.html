The example data set

The Lipsia-feeds package contains an example MRI data set to illustrate data processing in Lipsia. This example data set includes 3D high-resolution anatomical data and two functional data sets originating from a resting state experiment and a finger tapping experiment. Both anatomical and functional data are available in the Lipsia format.

For using the example MRI/fMRI data set, the Lipisa-feeds package (lipsia-feeds...rpm) must be installed.

Note that Lipsia is using the natural convention, i.e. L=L and R=R. Therefore, in the Lipsia data format the anatomical and functional data are stored in this natural convention. In contrast, in the Analyze files the data are stored in the radiological convention, i.e. L=R and R=L.

The example 3D high-resolution anatomical data set

The example data first contains an individual 3D high-resolution anatomical data set. The data has a resolution of 256x256x192 voxels (192 sagittal slices). The voxel resolution is 1x1x1mm (i.e. isotropic). The data set is available in the Lipsia data format. This file has the extension '.v'. In order to demonstrate the data conversion.

structural.v.gz: 3D anatomical data set in the Lipsia format

Before use, files with the extension '.gz' have to be uncompressed, e.g.

gzip2 -d structural.v.gz

The anatomical Lipsia data set can then be viewed using the command

vlview -in structural.v

The example functional resting state data set

The first example data set contains functional MRI data from a resting state experiment (i.e. an experiment without any stimulation). The data consists of 120 volumes (scans) with a repetition time of 2.3 seconds. Thus, the length of this experiment was 460 seconds. The volumes consists of 34 (axial) functional slices with an in-plane resolution of 3x3mm (64x64 voxel) and a slice thickness of 4 mm. The data set is available in the Lipsia data format. This file has the extension '.v'

rs_data.v.gz: 34 functional slices in the Lipsia data format

Before use, files with the extension '.gz' have to be uncompressed, e.g.

gzip -d rs_data.v.gz

The data set can be viewed using the command

vqview -in rs_data.v

The example functional data set

The example data set further contains functional MRI data from a finger tapping experiment. The data consists of 120 volumes (scans) with a repetition time of 2 seconds. Thus, the length of the experiment was 240 seconds. The experiment consists of three 40 seconds finger tapping blocks. Six functional slices (axial) with an in-plane resolution of 3x3mm (64x64 voxel) and a slice thickness of 4.5mm were measured. The data set is available in the Lipsia data format. This file has the extension '.v'.

data.v.gz: 6 functional slices in the Lipsia data format

slicetime.txt: Slice timing information for Analyze data

design.txt: Design file (3 finger tapping blocks a 40s)

Before use, files with the extension '.gz' have to be uncompressed, e.g.

gzip -d data.v.gz

The data set can be viewed using the command

vqview -in data.v

Note that you either can use 'vsview' in lieu of 'vqview' to show functional data.
