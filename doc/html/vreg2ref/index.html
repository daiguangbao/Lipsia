LIPSIA     Normalization to a reference image
vnormalize

Background
The program vnormalize is a little wrapper script which combines the applications valign3d and vdotrans3d. It also has a very simplified user interface.
Actually it is designed to normalize functional images to a reference image. If you have an anatomical scan you can use this. But it is also possible to normalize only specifying a functional image.

Normalization of your functional data using an anatomical image
In most cases you scan your functional data in combination with an anatomical image. The call to normalize your functional data to the MNI-brain using your anatomical image looks like:

vreg2ref -fun functional_data.v -ana anatomical_data -out normalized_functional_data.v

Of course you can specify your own reference image:

vreg2ref -ref your_reference_image.v -fun functional_data.v -ana anatomical_data -out normalized_functional_data.v
Normalization of your functional data without an anatomical image
Sometimes you only have the functional data for normalization to a reference image. The call for this looks like:

vreg2ref -fun functional_data.v -out normalized_functional_data.v

So all you have to do is to omit the -ana parameter. vreg2ref automatically changes the way of registration to the reference image. If you have not specified a reference image vreg2ref automatically uses the Lipsia MNI-brain. If you want to use your own reference image the call looks like:

vreg2ref -ref your_reference_image.v -fun functional_data.v -out normalized_functional_data.v

Parameters of 'vreg2ref':
-help
Prints usage information.
-ref
Fixed image file. Default: /usr/share/lipsia/mni.v
-ana
Your anatomical image. Default: None
-fun
Your functional image. Default: None
-out
Your normalized image. Required
